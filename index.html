Projects

  • Image Captioner

    This project generates captions for input images. The module is designed using a pre-trained Inception model, GloVe word embeddings, and LSTM to process and generate the captions. It was trained over the Flicker8k dataset, which contains photos and their related captions and will be evaluated by BLEU scores. Available at GitHub.

  • Sentiment Analysis and Aspect Detection

    This project predicts sentiment, polarity, and aspect of a given review of an input cellphone. Compared the normal review and its negated form mentioned in this paper. Models that used negated text data showed average 2% improvement on their F1-scores over the normal ones. Best Model got F1-Score of 83%. An interface was implemented using Streamlit, which contains the best sentiment classifier, and the interface detects the reviews aspect using LDA, and calculates the polarity using probability thresholds. Available at GitHub.

  • Language Model

    Designed a Language Model which predicts the next word of an input sequence. This model is created from scratch, and I trained it over a dataset containing unigrams, bigrams, and trigrams. I also implemented three kinds of smoothing: None, Laplace, and Kneser-Kney. Word prediction is done using probability distribution of previous words. Every possible model was evaluated using word-error-rate. The model with trigrams and Kneser-Ney smoothing had the best result, 1.24 WER. Available at GitHub.

  • Ad-Recommender

    Built a model that recommends merchandise for an input text, based on the text themes, for online advertising. The dataset contained both labeled and unlabeled data. I used Semi- Supervised learning techniques to train a classifier for the desired labels and crawled a list of goods using Scrapy from Digikala online shop related to the labels. Used different basic models and different probability thresholds for semi-supervised procedure. Best model was an SGD model that had F1-score of 87.5% using 80% threshold for unlabeled datapoints. Available at GitHub.

  • Condition Classifier

    A group project, our intention was to design a model that receives an online medical review and tries to predict the writer’s illness, and in some cases recommends medicine that might be helpful. My role was to try different models to find the best one. SVC had the best performance. We Used a dataset from Kaggle containing reviews written in an online forum about illness symptoms and conditions, and in some cases, the medicine used for treatment to build our classifier. Available at GitHub.

  • Information Retrieval

    Created a search engine over a colossal dataset of Persian news, which answers to incoming queries. I implemented this engine in two phases: First checked heaps’ and zif’s laws’ accuracy on tokens of the corpus. Later, Implemented the search engine using TF-IDF and document vectors and finally added a simple UI for incoming queries. available at GitHub.

  • Database

    Designed an ETL, which creates a duplicate of the database for recovery and updates the backup database whenever a change happens in the source database. Implemented this project using psycopg2 Python library. Available at GitHub.

  • Principle of Software Design

    A group project. We Implemented an interface that recommends music based on a picture of a person’s face. My role was to implement backend features including user authen- tication, data transfer, fetching metadata of Spotify’s music tracks, and research about Face APIs. We used AzureAPI to classify the mood of the picture, and Spotify Analytics API to understand the content of music tracks. We used Django for backend and Reactjs for frontend. Available at GitLab.
